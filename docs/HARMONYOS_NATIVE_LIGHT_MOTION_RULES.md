# HarmonyOS 原生轻动效规则

这套规则沉淀自“一局”当前版本，目标是让 HarmonyOS NEXT / ArkUI 应用保持原生、克制、轻盈的触感。后续开发其他鸿蒙 App 时，可以直接作为动效基线复用。

## 1. 启动体验

- 启动过渡优先交给系统 `startWindowBackground` 和桌面图标进入应用的原生动效承接。
- 首页不额外做冷启动动画，不再叠加 logo 放大、页面淡入、闪屏图、引导图等二次启动内容。
- 如果不希望启动瞬间屏幕中央额外显示 App 图标，可将 `startWindowIcon` 指向透明资源；桌面图标仍由 ability `icon` 控制。
- `startWindowBackground`、窗口背景、窗口容器背景和首页页面背景必须保持同色。
- 当前推荐背景色：`#F1F3F5`。

## 2. 页面背景

- 页面根节点、窗口背景、启动窗背景使用同一浅灰底色，避免状态栏、手势导航栏、页面主体之间出现断层。
- 不要为了“沉浸感”把普通应用窗口设为透明，除非已经在真机验证状态栏和手势导航栏区域不会透出系统层。
- 内容区域使用柔白卡片承载信息，页面 section 不做强主题渐变或复杂装饰。

## 3. 卡片按压

- 所有主要内容卡片都需要按压反馈，即使卡片本身没有跳转。
- 按下缩放：`{ x: 0.96, y: 0.96 }`。
- 松手恢复：`{ x: 1, y: 1 }`。
- 动画时间：`150ms`。
- 动画曲线：`Curve.FastOutSlowIn`。
- 按下时阴影同步收敛，通常减少 `radius` 和 `offsetY`，形成“被轻轻按进页面”的触感。

参考参数：

```ts
private pressableCardScale(cardKey: string): ScaleOptions {
  return this.pressedCardKey === cardKey ? { x: 0.96, y: 0.96 } : { x: 1, y: 1 };
}

private pressableCardShadowRadius(cardKey: string, radius: number = 18): number {
  return this.pressedCardKey === cardKey ? Math.max(radius - 8, 8) : radius;
}

private pressableCardShadowOffsetY(cardKey: string, offsetY: number = 8): number {
  return this.pressedCardKey === cardKey ? Math.max(offsetY - 4, 3) : offsetY;
}
```

## 4. 按钮和胶囊

- 按钮、小胶囊、列表操作、筛选入口、搜索入口、导航项都需要按压缩小反馈。
- 按钮缩放建议同卡片保持一致：`0.96`。
- 反馈必须轻，不使用弹跳、旋转、闪烁、夸张放大。
- 同一业务层级的按钮反馈时间和曲线保持一致。

## 5. 返回按钮

- 返回按钮使用圆形符号按钮，不使用纯文本 `<`。
- 推荐实现：`ButtonType.Circle` + `SymbolGlyph($r('sys.symbol.chevron_left'))`。
- 推荐尺寸：`40vp x 40vp`。
- 背景使用半透明白色和轻玻璃模糊；保持圆形，不做大块矩形按钮。
- 返回按钮也需要按压缩小反馈。

## 6. 边缘返回手势

- 所有带返回按钮的整页详情，都支持从屏幕左边缘向右滑返回。
- 详情页必须接入 ArkUI 页面级 `onBackPress()`，优先消费系统返回键和左缘系统返回手势；只有没有内部详情页可返回时，才允许系统退出 App。
- 不要只依赖自定义 `PanGesture` 实现边缘返回；真机左缘系统手势优先级更高，未接入 `onBackPress()` 时会直接退出当前 Ability。
- 带返回按钮的详情页应作为覆盖在主页面之上的整页层，从右侧 `TransitionEdge.END` 进入，返回时向右侧退出，保持与主页面横向排列的空间关系。
- 详情页根节点必须设置与首页一致的页面背景色，转场过程中不能露出透明系统层。
- 推荐起滑区域：左侧 `24vp` 内。
- 推荐横向触发阈值：`72vp` 以上。
- 为避免影响纵向滚动，限制纵向偏移，例如 `Math.abs(offsetY) <= 80`。
- 该手势只用于整页返回，不用于 `bindSheet` 内部临时任务。

## 7. 底部导航

- 底部导航使用单一悬浮玻璃容器，不叠多层复杂背景。
- 每个 Tab 固定尺寸，推荐 `60vp x 60vp`。
- 选中态主要通过颜色和图标轻微上移表达，不使用大面积移动背景块。
- 图标和文字必须显式居中。

## 8. Sheet

- 临时编辑任务优先使用 ArkUI `bindSheet`。
- Sheet 打开时，底层页面可以轻微缩小并降低透明度。
- 推荐底层变化：`scale(0.985)`、`opacity(0.66)`。
- Sheet 内按钮、列表行和可点击入口仍然保留统一按压反馈。

## 9. 滚动和列表

- 主页面、详情页、流水列表优先使用 `EdgeEffect.Spring`，保留鸿蒙原生弹性。
- 可点击列表行需要按压反馈。
- 删除、撤回等横向操作优先使用系统 `swipeAction`，避免自造复杂拖拽。
- 顶部搜索入口可使用 Header Transition：点击搜索栏或页面上滑时，标题等顶部内容向上平移并淡出，搜索栏保留在顶部并平滑切换为输入态。
- Header Transition 不应跳转到右侧详情页；它是同页搜索态，底部导航可临时隐藏，返回键用于退出搜索态。
- Header Transition 建议统一使用 `260ms-280ms` + `Curve.FastOutSlowIn`，标题淡出上移、搜索栏尺寸/阴影变化、搜索内容出现应同曲线联动，避免一个元素先跳完另一个才动。
- 搜索空态或引导语不要裸露为纯文本，应使用轻量说明卡片承载，例如浅色底、细边框、标题和说明文字。
- 当搜索需要自动聚焦、拉起键盘、取消按钮、独立返回和屏蔽主 Tab 横向切换时，应升级为整页搜索临时层；临时层可以复用详情页承载结构，但转场宜使用轻量淡入/上浮，不使用普通详情页的右侧横推。
- 搜索临时层从列表页搜索框进入时，应优先做伪共享元素过渡：新页面搜索框从原搜索框的大致位置上移到顶部，同时背景由轻透明过渡到完整页面底色，取消按钮和内容稍后淡入。
- 伪共享搜索框退出时，取消按钮、搜索框右侧留白、阴影、圆角、宽度和纵向位置必须使用同一 `260ms-280ms` 曲线同步收拢；底层原搜索框保持隐藏到临时层卸载后再显示，避免出现两个搜索框或宽度突然跳变。

## 10. 图标居中

- 固定尺寸图标、头像字、符号按钮、空态图标必须显式居中。
- 搜索、返回等常见符号优先使用 `SymbolGlyph`，避免用文本字符的字体基线造成视觉偏上或偏下。
- 横向居中：`textAlign(TextAlign.Center)` 或容器 `alignItems(HorizontalAlign.Center)`。
- 纵向居中：固定高度文本配合 `lineHeight`，或使用 `Stack({ alignContent: Alignment.Center })`。
- 不要只依赖字体默认行高，真机字体渲染可能导致视觉偏上或偏下。

## 11. 验证规则

- 涉及窗口背景、启动窗、状态栏、手势导航栏、全屏/沉浸式能力的改动，必须真机验证。
- 模拟器验证只能作为第一层编译和基础交互检查，不能替代真机对系统栏、手势区域、启动过渡的验证。
- 任何参考其他 App 的窗口或启动实现时，只能迁移明确适合当前 App 的部分；透明窗口、全屏布局、状态栏避让等能力必须单独验证。
