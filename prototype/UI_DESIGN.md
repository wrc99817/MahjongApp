## 当前界面形态

当前 ArkUI App 已从早期“账本/记一局/统计”调整为四个主 Tab：

- 牌局：当前牌局工作台。
- 统计：全部、历史牌局两个分段。
- 牌友：搜索框、牌友列表和搜索态。
- 我的：我的牌局统计、关于和清空所有数据入口。

底部为四项悬浮玻璃胶囊导航。无进行中牌局时，牌局页通过一张新建牌局卡片承载主操作；点击卡片跳转到新建牌局页。编辑牌局使用 ArkUI `bindSheet` 底部半屏 Sheet，历史牌局详情使用整页详情。

Web 原型画布的页面摆放顺序应按完整功能流程组织：牌局空状态 -> 新建牌局 -> 牌友抽屉选择态 -> 进行中牌局操作 -> 编辑/结算 -> 结算完成 -> 统计与历史详情 -> 牌友与搜索 -> 我的。

## 设计方向

雀账 V1.0 主打离线模式的快速记账体验。视觉上应保持鸿蒙原生、轻盈、克制、清晰，不做跨端 H5 管理后台式堆叠。

- 页面结构清晰，信息密度适中，核心账面和输入操作优先露出。
- 使用 ArkUI 原生控件、圆角层级、轻量阴影、模糊材质和自然动效。
- 关键操作集中在屏幕下半区，适合单手操作。
- 金额信息用红/绿表达输赢，但颜色要降低刺激感。
- 视觉可以有麻将气质，但不走复古牌桌风，也不做厚重纹理。
- V1.0 以本地离线为主，界面文案和入口要让用户感到“本地、隐私、可控”。

## 样式特色规则

雀账的视觉特色可以概括为“轻玻璃导航 + 柔白卡片 + 金额语义色 + 原生短动效”。后续新增页面、组件和特效都必须先匹配这套语言，再考虑局部创新。

### 1. 轻玻璃感只用于导航和系统级浮层

- 底部导航、Sheet 背景和需要覆盖页面的轻浮层可以使用玻璃模糊。
- 普通内容卡片不使用大面积玻璃，不使用高饱和渐变，不使用装饰性光斑。
- 玻璃层必须半透明、浅色、低对比，保持“浮在页面上”的感觉，而不是变成厚重遮罩。
- Sheet 弹出时，底层页面可以轻微缩放和降低透明度，强化空间层级。

### 2. 卡片负责分组，不负责炫技

- 卡片只承载一组明确任务：账面、输入、流水、统计、列表、风险操作。
- 一个信息组只用一层卡片；不要在大卡片里继续套大卡片。
- 卡片圆角保持 22vp 到 24vp，阴影保持轻柔，避免强投影和悬浮堆叠。
- 内容密度优先服务记账效率，页面不要做营销页式大留白。

### 3. 蓝色是操作，红绿是结果

- `#2F63F4` 蓝色只用于主按钮、选中态、导航强调和可点击入口。
- `#2E8B57` 绿色只表示正向金额、赢、成功或安全状态。
- `#D94C45` 红色只表示负向金额、输、风险或清空数据。
- 同一屏幕内不要同时堆叠多种强调色；金额和按钮同时出现时，按钮优先用蓝色，金额保留红绿语义。

### 4. 金额是主角，装饰退后

- 账面、输赢、应收应付、结算金额应使用更高字重和更稳定的数字排版。
- 正负金额必须通过符号和颜色共同表达，例如 `+56` / `-20`。
- 金额附近的说明文字保持 12fp 到 14fp，避免抢走金额焦点。
- 任何特效都不得遮挡、移动或弱化金额结果。

### 5. 动效只表达状态变化

- Tab、分段控件、按钮按压、搜索态、Sheet、页面进退场可以有动效。
- 动效时长默认 160ms 到 260ms，使用 `Curve.FastOutSlowIn` 或接近鸿蒙原生的曲线。
- 按钮、可点击小胶囊、可点击列表行和卡片按压只做轻微缩放、阴影收敛或透明度变化，不做弹跳、旋转、闪烁；所有可点击控件需要使用统一的轻微缩小按压反馈，后续若抽出公共 Button、Capsule、ListRow 或 Card 组件，应在公共组件层统一实现。
- 金额变化可以使用短暂高亮或轻微淡入，但不做数字翻滚、粒子、爆炸式庆祝。
- 列表新增、作废、修改流水时使用淡入淡出或轻微位移，保持可追溯感，不制造“记录消失”的错觉。

### 6. Sheet 是主要编辑形态

- 编辑牌局、选择牌友、流水修正等临时任务优先使用 ArkUI `bindSheet` 半屏形态。
- Sheet 顶部必须有清晰标题，内容按表单或列表自然排列，主操作放在底部内容末端。
- Sheet 出现时允许背景缩放到约 `0.985` 并降低透明度；关闭后必须恢复。
- 不使用自绘全屏弹窗模拟 Sheet；能用 ArkUI 原生能力时优先用原生能力。

### 7. 列表和历史强调“可扫读”

- 流水、历史牌局、牌友列表保持行高稳定，左侧信息、右侧金额或状态。
- 每条记录最多两级文字：标题和说明；不要在列表行里塞复杂按钮组。
- 可操作记录使用轻量入口，例如长按、更多按钮或详情 Sheet，避免每行铺满操作按钮。
- 作废记录如需要显示，应有清晰作废状态，但不参与账面和统计计算。

### 8. 麻将气质克制表达

- 可以使用“牌局、牌友、封账、入场、流水”等领域文案建立麻将语境。
- 可使用少量牌桌绿色、朱砂红、浅金色作为辅助，但不得替代当前蓝灰主系统。
- 不使用麻将牌纹理、木纹、深绿赌桌背景、大红大金等强主题装饰。
- 图标优先线性、简洁、近原生风格，不做拟真麻将牌。

### 9. 空态和风险态要温和

- 空态用于引导下一步，例如新建牌局、暂无流水、暂无历史，不使用夸张插画。
- 风险操作使用红色浅底和二次确认，文案直接说明后果。
- 本地隐私、离线保存、清空数据等敏感点要表达“可控”，不要制造警告感。

### 10. 新增特效验收口径

新增任何视觉特效前，需要满足以下条件：

- 不影响核心记账链路的速度和可读性。
- 不遮挡金额、输入框、按钮、底部导航和 Sheet 主操作。
- 动效结束后状态明确，不要求用户猜测当前页面在哪里。
- 与现有颜色、圆角、阴影、玻璃和时长规则一致。
- 在小屏、字体放大和键盘弹出场景下不重叠、不跳动、不丢失操作入口。

## 全局布局

屏幕分为三层：

1. 顶部状态与标题层。
2. 中部可滚动内容层。
3. 底部悬浮导航层。

页面内容底部必须预留足够空间，避免被悬浮导航遮挡。当前主页面可参考 `Index.ets` 中的 `Screen()` 结构：外层 `Stack` 放内容和底部导航，四个主页面使用 `Swiper` 承载左右滑动切换，每个 Tab 内部保留独立 `Scroll`，底部按页面状态隐藏或展示导航。主页面标题只展示大标题，不展示标题上方的小字说明。

## 底部悬浮导航

导航结构：

```text
牌局    统计    牌友    我的
```

视觉规则：

- 高度约 72vp。
- 宽度约为屏幕 88% 或左右保留 20vp 边距。
- 底部距离 16vp 到 24vp，避开系统手势区域。
- 圆角 36vp，形成完整胶囊。
- 背景为更轻的半透明液态玻璃材质，使用 `BlurStyle.COMPONENT_ULTRA_THICK`，外层透明白、内层高光和柔和阴影叠加。
- 边框使用高亮半透明描边，并保留顶部细高光。
- 阴影柔和，避免厚重浮层感。
- 选中 Tab 使用轻色底和蓝色图标/文字。
- 未选中 Tab 的图标和文字统一使用灰色，不保留蓝色强调。

当前 App 已使用蓝色作为主操作色；如后续回到麻将朱砂红主按钮，需要同步调整页面内主按钮、强调色和截图。

## 颜色系统

当前 ArkUI 实现使用偏鸿蒙原生的浅蓝灰系统：

```text
页面背景：#F3F6FC
内容底色：#FFFFFF
柔和底色：#EDF2FA
导航玻璃：#B8FFFFFF
主文字：#1F2329
次文字：#69707A
分割线：#E4EAF3
主行动蓝：#2F63F4
蓝色浅底：#EAF1FF
赢/正向绿：#2E8B57
输/风险红：#D94C45
```

设计约束：

- 不要把页面做成单一蓝色或单一红色主题。
- 蓝色用于主操作、选中态和导航强调。
- 红/绿只服务金额、风险和输赢语义。
- 卡片背景保持白色或极浅底色，减少视觉噪声。

## 字体与层级

- 页面标题：24fp 到 28fp，Bold。
- 模块标题：16fp 到 18fp，Bold。
- 正文：13fp 到 16fp。
- 金额数字：18fp 到 28fp，根据层级加粗。
- Tab 与胶囊内文字：10fp 到 13fp，保持紧凑。
- 不使用夸张大标题，优先让表格、流水、金额和输入控件可扫读。

## 圆角与卡片

当前界面以圆润原生风格为主：

- 页面卡片：22vp 到 24vp。
- 输入框：13vp 到 21vp。
- 分段按钮：15vp 到 18vp。
- 头像/字母标识：圆形或 14vp 到 18vp 圆角。
- 底部导航：36vp。
- 新建牌局卡片内图标按钮：20vp 到 24vp 图标，外层容器 52vp 到 56vp。

卡片可以用于独立信息组，例如当前账面、记账输入、流水、统计块。避免在卡片内部继续套大型卡片。

## 动效

动效要短促、自然，接近鸿蒙原生手感：

- Tab 切换：选中态轻微过渡，`Curve.FastOutSlowIn`，约 180ms 到 220ms。
- Sheet 出现：底部弹出，背景内容轻微缩放和降透明度。
- 新建牌局页进入：从牌局页卡片跳转到独立页面，顶部返回箭头回到牌局页。
- 按钮按压：使用 ArkUI `stateEffect(true)` 并配合轻量缩放；可点击小胶囊和列表行也需要同等缩放反馈。
- 搜索态：牌友页进入搜索时隐藏底部导航，顶部搜索栏固定。

## 牌局页

牌局页是当前 App 的核心工作台。无进行中牌局时，标题使用 `牌局`，标题下只展示一张新建牌局卡片，文案为 `点击此处开启一场牌局吧`，点击后跳转到新建牌局页。

新建牌局页使用整页表单，不使用底部 Sheet。顶部标题为 `新建牌局`，标题左侧放返回箭头；页面内按“引导说明 / 牌局信息 / 牌友和入场金额 / 开始记账”分组，输入项间距要比抽屉表单更舒展。牌友 1 固定展示当前用户“我”，不使用输入框或下拉框；牌友 2 到牌友 4 的姓名控件由输入区和右侧箭头区组成：点击输入区进入正常文本输入状态并显示光标；点击右侧箭头才弹出底部牌友选择抽屉。四行左侧使用 `牌友1` / `牌友2` / `牌友3` / `牌友4` 文字标签，不使用姓名缩写圆形图标。抽屉只展示可滚动的非本人已存在牌友，不展示新建选项和完成按钮，暂无牌友时展示空态提示，且覆盖在页面上方，不应挤开下方表单。输入已存在姓名时自动选择已有牌友，输入新姓名时以输入框当前姓名创建本地牌友；新建页不预填模拟牌友或默认入场金额，入场金额需由用户填写。

有进行中牌局时，信息优先级如下：

1. 当前牌局标题。
2. 牌友和入场金额。
3. 编辑/结算入口。
4. 当前账面。
5. 记账输入。
6. 进出账历史。

有进行中牌局时，`牌友和入场金额` 使用等宽 1x4 横向布局，四个牌友块占满卡片内容宽度，并和下方编辑/结算按钮左右对齐；牌友名和入场金额上下排列，避免胶囊散落导致视觉不齐。

记账输入使用三段分段控件：

- 单家收分。
- AA 开销。
- 手动调整。

记账输入以清晰录入为主，单家收分不展示预流水，保存时再生成真实流水；AA 开销和手动调整也不依赖预流水作为主要反馈。

结算页的应收应付区域按牌友展示最终账面：左侧只显示牌友名称和入场金额，右侧金额必须带正负号并使用金额语义色；具体“谁转给谁”不放在应收应付行内，而是进入下方“最佳转账路径”卡片。最佳转账路径用最少转账步骤展示付款人、收款人和金额。

结算完成页用于承接封账后的情绪反馈，保持克制但应有明确完成感：使用柔白结果卡片展示封账完成、赢家文案、赢的金额和统计归档提示；卡片进入时轻微上移滑入，金额从 0 以较长、丝滑的缓动计数到最终值；返回后牌局页恢复为空状态。

## 统计页

统计页由两个分段组成：

- 全部。
- 历史牌局。

全部用于快速复盘，不做复杂报表，卡片结构为：总览、最近封账、走势、胜率最高牌友。统计总览卡中的同级数据必须统一 label/value/unit 三段高度，让三个核心数值落在同一水平线上。走势折线图需要显示日期标签，避免只有孤立点线。历史牌局用于进入单局详情，历史牌局区域不再包一层父卡片，直接以每局牌局作为独立卡片；历史牌局卡片需要展示日期、参与人、赢家、输家和输赢金额。历史详情页以卡片分区展示“牌局概览”“最终账面”和“进出账流水记录”，概览卡最上方应显示大赢家、大输家、牌局日期、参与人名称和入场金额总览；其他信息只做轻量辅助。封账后的牌局结果必须进入统计、历史牌局和牌友同桌数据；后续把演示统计替换为真实仓储计算时，视觉结构保持稳定，只替换数据来源。

## 牌友页

牌友页用于离线模式下的本地牌友表现查看，并为后续在线模式预留入口。

当前保留：

- 固定标题。
- 搜索框。
- 牌友列表。
- 搜索态、结果态、空态。

搜索态要轻量，不弹复杂浮层；进入搜索时隐藏底部导航，取消后恢复。

## 我的页

我的页当前保留三类内容：

- 我的牌局所有统计情况，统计卡片不显示“所有统计”标题。
- 关于。
- 清空所有数据按钮。

清空所有数据是风险操作，必须使用红色标识；后续 App 实现时需要二次确认。暂不展示本地备份、导出账本、在线模式设置、通知设置和主题切换等入口。

## Sheet 规范

编辑牌局使用底部 Sheet：

- 顶部显示清晰标题。
- 输入项优先使用原生 `TextInput`。
- 主按钮固定在 Sheet 底部内容末端。
- 键盘弹出时使用 `SheetKeyboardAvoidMode.TRANSLATE_AND_SCROLL`。
- Sheet 背景保持白色或半透明白，不做深色遮罩卡片堆叠。

## 文档合并建议

当前不建议只依赖 Web 原型图开发。截图能说明视觉结果，但不能完整表达金额校验、分录平衡、Sheet 状态、搜索态、历史详情、统计口径和数据模型。

推荐保留两份轻量文档：

- `UI_DESIGN.md`：只写视觉基线、设计约束和组件气质。
- `PROTOTYPE_DEV_SPEC.md`：只写页面结构、状态逻辑、业务流程和数据结构。

如果后续继续精简，可以把本文件合并进 `PROTOTYPE_DEV_SPEC.md` 的“视觉规范”章节，然后删除 `UI_DESIGN.md`。不建议两个文件都删除。
