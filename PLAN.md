# 纯血鸿蒙麻将记账 App 方案

## Summary
第一版定位先收口为“离线麻将账本”：线下麻将玩家可以在本机创建牌局账本，手动录入每局四人输赢和共同开销，完整查看明细、总账、统计和结算结果。好友群组同步账本作为后续在线版本继续迭代。

技术底座采用纯血鸿蒙：HarmonyOS NEXT Stage 模型、ArkTS、ArkUI；本地优先存储加云端同步。参考官方方向：[Stage 模型](https://developer.huawei.com/consumer/cn/arkui/arkui-stage/) 和 [AppGallery Connect 云数据库](https://developer.huawei.com/consumer/cn/agconnect/cloud-base/)。

## Key Features
- 本地账本：创建本机离线账本，维护牌友昵称、牌局列表和封账历史。
- 牌局记录：创建一场牌局，设置玩家、底分/备注，按局录入每个玩家输赢金额。
- 自动校验：每局四人输赢合计必须为 0；支持撤回/删除，并保留操作记录；流水修改暂缓。
- 结算看板：按牌局、按日期、按玩家汇总总输赢、场均输赢、最大单局、最近走势。
- 离线可靠性：关闭或重启 App 后保留牌友、牌局、流水、操作记录和封账历史。
- 完整历史：进行中流水、历史牌局和历史详情支持完整查看，不因列表截断丢失记录。
- 后续在线协作：登录、邀请成员、云同步、分享导出和权限控制放入 V1.5 / V2。

## Architecture
- 客户端：ArkTS + ArkUI，采用 Stage 模型；页面包括首页账本、牌局详情、录入局数、统计、成员管理、设置。
- 状态管理：页面内用 ArkUI 状态，跨页面共享用轻量 Store/Repository 分层，避免 UI 直接访问数据库。
- 本地数据：V1.0 使用 HarmonyOS 本地关系型数据库或合适的本地持久化能力保存用户、账本组、成员、牌局、局记录和操作日志，保证离线可用。
- 云端同步：V2 使用 AppGallery Connect 认证服务做登录，云数据库做群组账本、多端同步和权限控制。
- 数据模型核心：
  - `User`：用户基础信息。
  - `Group`：麻将群/账本。
  - `GroupMember`：成员角色与权限。
  - `GameSession`：一次牌局。
  - `RoundRecord`：单局输赢明细。
  - `LedgerOperation`：删除、撤回等审计记录，后续如恢复修改功能再扩展修改记录。
- 操作日志：本地写入后生成操作记录；撤回/删除流水必须保留可追溯记录。云同步状态和冲突处理留到后续在线版本。

## Iteration Plan
- V1.0 离线版：创建/编辑牌局、三种记账输入、合计校验、流水、结算、统计、牌友页、我的页、本地持久化、流水撤回/删除、完整历史与流水查看；流水修改暂缓。
- V1.5：常用玩家、历史筛选增强、账单分享、离线同步状态提示、本地备份、通知设置和主题切换。
- V2 在线版：登录、建群、邀请加入、云备份、多群管理、AppGallery Connect 云同步、权限控制、高级统计、导出 Excel/PDF、规则模板市场。
- V3 专业版：地方玩法辅助计算、战绩排行榜、AI 复盘摘要、桌面卡片/服务卡片。

## Test Plan
- 单局金额校验：四名玩家输赢合计不为 0 时禁止保存。
- 本地持久化：创建牌友、牌局、流水和封账历史后重启 App，数据仍然完整可见。
- 流水修正：撤回/删除流水后，账面、统计、结算、历史详情和操作记录保持一致。
- 完整历史：长牌局多笔流水、多场已封账历史都能完整查看。
- 后续在线场景：多人同步、恢复网络后同步且不重复、权限控制进入 V2 测试范围。
- 数据可靠性：删除/撤回记录可追溯，统计结果与明细一致。
- 鸿蒙适配：手机、折叠屏、平板基础布局适配；深色模式和字体放大不遮挡。

## Assumptions
- 第一版采用“手动输赢记账”，不做完整番型算分。
- 第一版先做可独立使用的离线版本，好友群组同步进入后续在线版本。
- 登录和云同步后续优先使用华为生态能力，保持纯血鸿蒙技术路线。
- 商业化能力暂不进入 MVP，只预留账号、会员状态和云端配置扩展点。
